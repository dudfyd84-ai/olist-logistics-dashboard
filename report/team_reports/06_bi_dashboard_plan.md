# [BI] 대시보드 시각화 기획 리포트

**작성자**: BI / 대시보드 시각화 전문가
**주제**: 5개 파트의 분석 결과를 통합 전달하는 종합 비즈니스 대시보드 구현안

## 1. 대시보드 구현 개요 및 기술 스택
- **사용 기술**: Python `Streamlit`, 시각화는 인터랙티브 기능이 뛰어난 `Plotly` 독점 사용.
- **목적**: 경영진이 한 화면에서 매출, 배송 품질, 고객 유지 지표를 모니터링하고 실무자가 상세 데이터를 필터링할 수 있도록 구성.

## 2. 레이아웃 설계 (4개의 탭 구조)

### 탭 1. 종합 상황판 (Executive Summary)
- PM 파트의 비즈니스 인사이트 요약
- DAU/MAU 및 전체 누적 매출 메트릭 패널
- 전체 이탈률(취소율) 및 정시 배송률 도넛 차트

### 탭 2. 가설 검증 통계 (지연일 vs 리뷰 점수)
- DA(데이터 분석가)의 가설 검증 결과 (Scatter Plot 및 OLS 회귀선)
- 배송 지연 일수 증가에 따른 평점 하락폭 시각화
- 가설 참/거짓 판단 메시지 배너

### 탭 3. 지역별 물류(SCM) 현황 지도
- SCM 파트의 분석을 토대로, 위경도 좌표를 활용한 브라질 주(State)별 매핑(Map)
- 거점에 따른 지연율 히트맵(Heatmap)을 지도 위에 레이어 오버레이 방식으로 표시 (Plotly Mapbox)

### 탭 4. 카테고리/마케팅 수익성 분석
- 마케팅 파트의 AOV 및 운임 비중(Freight Ratio) 박스플롯 비교
- 상위/하위 셀러를 구분한 월간 코호트 리텐션(고객 수 & 매출액) 매트릭스 시각화

## 3. 구현 유의사항
- 약 10만 건 이상의 대규모 데이터 병합 결과물이므로 Streamlit 내부에서 `@st.cache_data`를 활용하여 필터링 속도 저하를 방지.
- 색상 테마는 신뢰감을 주고 가독성이 높은 컬러 팔레트를 채택하되, 지연(Late) 및 하락 지표는 붉은색 톤으로 시각적 경각심을 줌.
